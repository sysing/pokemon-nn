# v0.6 — RandomResizedCrop Augmentation

**Date**: 2026-06-14
**Dataset**: 934 images, 10 types
**Device**: M4 MacBook Air (MPS)

## Changes from v0.5

- Replaced `Resize(224,224)` with `RandomResizedCrop(224, scale=(0.7, 1.0), ratio=(0.9, 1.1))` in training transforms
- Model sees randomly cropped/zoomed versions of cards each epoch — learns scale/position invariance
- Added weight norm logging (`weights.csv`) and live weight visualization panel
- Added `--resume` flag to continue training from a checkpoint

## Key Findings

- **+0.6pp** over v0.4 (91.9% → 92.5%)
- **Dragon jumped from 89.5% → 100%** — the crop forces the model to focus on art features (dragon silhouette) rather than card layout/border position
- **Fighting improved from 87.5% → 93.8%** — large gain
- **Colorless regressed from 80.0% → 73.3%** — without a strong colour theme, random cropping confuses the model. This is the hardest type by a wide margin.
- **Training takes longer to converge** — epoch 1 val acc dropped from 70% to 60% because the model has to work harder on each image variant
- RandomResizedCrop is a net positive: gains on Dragon and Fighting outweigh the Colorless regression
