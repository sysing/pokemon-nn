# v0.1 — Baseline

**Date**: 2026-06-14
**Dataset**: 855 images, 10 types (Colorless, Darkness, Dragon, Fighting, Fire, Grass, Lightning, Metal, Psychic, Water)
**Device**: M4 MacBook Air (MPS)

## Configuration

| Parameter | Value |
|-----------|-------|
| Model | PokemonTypeCNN (4 conv blocks, ~600K params) |
| Optimizer | AdamW, lr=1e-3, weight_decay=1e-4 |
| Schedule | ReduceLROnPlateau (factor=0.5, patience=3) |
| Loss | CrossEntropyLoss with class weights |
| Augmentation | RandomHorizontalFlip, RandomRotation(5°), ColorJitter(0.15) |
| Batch size | 32 |
| Epochs | 7 (best at epoch 6, early stop) |

## Results

**Overall val accuracy**: 86.5%

| Type | Accuracy | Samples |
|------|----------|---------|
| Fighting | 100.0% | 17 |
| Dragon | 100.0% | 5 |
| Lightning | 100.0% | 21 |
| Water | 100.0% | 18 |
| Psychic | 94.7% | 19 |
| Darkness | 92.3% | 13 |
| Fire | 92.3% | 13 |
| Grass | 89.3% | 28 |
| Colorless | 76.5% | 17 |
| Metal | 35.0% | 20 |

## Insights

- **Color is the dominant signal**: The model achieves high accuracy on types with distinct color palettes (Water=blue, Fire=red, Grass=green).
- **Metal weakness**: Metal and Colorless cards share similar silver/grey backgrounds. The model over-relies on color and cannot distinguish them.
- **Class imbalance**: Dragon has only 21 total samples (5 in val split). The 100% accuracy is likely unreliable due to small sample size.
- **Hypothesis for v0.2**: Adding RandomGrayscale augmentation should force the model to learn non-color features (card layout, text placement, evolution box style) improving Metal classification.
