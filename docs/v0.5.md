# v0.5 — Curriculum Pretrain (Colorless + Metal first)

**Date**: 2026-06-15
**Dataset**: 934 images, 10 types
**Device**: M4 MacBook Air (MPS)

## Changes from v0.4

- **Two-phase curriculum training**: Phase 1 pretrains on Colorless + Metal only (binary), Phase 2 trains on all 10 classes
- Features learned in Phase 1 are transferred to Phase 2, classifier head is randomly initialized
- Same 4-channel architecture and label smoothing (0.1) from v0.4

## Hypothesis

Colorless and Metal are visually similar (both silver/grey). By pretraining on just these two classes in isolation, the model learns distinguishing features (card layout, text placement, attack box structure) *before* being distracted by 8 easy color-based classes.

## Insights

- **Hypothesis confirmed**: Metal hit 95.2% — our highest ever. Curriculum pretrain forced the model to learn Metal/Colorless distinguishing features without distraction.
- **Asymmetric transfer**: Metal improved (+4.7pp) but Colorless regressed (−6.7pp). Phase 1 features may be biased toward Metal-dominant patterns.
- **Lightning collapse** (−19pp) is the main regression. Lightning was 100% in v0.4. Possible cause: pretrained features from the binary task damaged the color pathway that Lightning relied on.
- **Net effect is neutral** (−0.5pp overall). Curriculum pretrain helps the hardest class pair but doesn't improve overall accuracy — it shifts where errors occur rather than reducing total errors.
- **Next direction**: Instead of full feature transfer, try *partial transfer* — only transfer early conv layers, or use a lower learning rate for transferred layers to preserve pretrained features while adapting to new classes.
