# v0.2 — RandomGrayscale Augmentation

**Date**: 2026-06-14
**Dataset**: 855 images, 10 types
**Device**: M4 MacBook Air (MPS)

## Changes from v0.1

- Added `RandomGrayscale(p=0.3)` to training augmentations — 30% of images are randomly converted to grayscale
- Training artifacts now saved to `runs/v0.2/`

## Hypothesis

Metal and Colorless cards share similar silver/grey color palettes. By randomly removing color during training, the model is forced to learn non-color features (card layout, text placement, evolution badges, attack box structure) that distinguish these types.

## Results

**Overall val accuracy**: 81.3% (↓5.2pp from v0.1)
**Best epoch**: 28

| Type | v0.2 | v0.1 | Δ |
|------|------|------|---|
| Colorless | 70.6% | 76.5% | −5.9 |
| Darkness | 100.0% | 92.3% | +7.7 |
| Dragon | 100.0% | 100.0% | — |
| Fighting | 64.7% | 100.0% | −35.3 |
| Fire | 76.9% | 92.3% | −15.4 |

| Water | 94.4% | 100.0% | −5.6 |

## Insights

- **Hypothesis confirmed**: Metal improved from 35% to 60% (+25pp). RandomGrayscale forces learning of layout/text features.
- **Trade-off**: Overall accuracy dropped 5.2pp because color is genuinely the strongest signal for most types.
- **Fighting collapse**: Fighting dropped hardest (−35pp) — its brown/tan color palette is its main distinguishing feature from the pokemontcg.io image set used.
- **Dragon sample size**: Dragon held at 100% but only has 5 val samples — not statistically meaningful.
- **Next direction**: Instead of random grayscale, try adding grayscale as a separate channel or using a multi-branch architecture that processes both color and grayscale.
